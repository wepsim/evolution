中央処理装置：メインメモリアクセス

M1 1ビット

データレジスタ（MBR）の更新に使用できる値を、メモリからの値または内部データバスからの値のいずれかで選択できます。

MH 1 bit

この信号を使用すると、次のハードウェア パフォーマンス カウンター (HPC) を選択できます。

  • 00: クロック サイクル数。.
  • 01: 累積 CPU 時間。.
  • 10: 累積消費電力.
  • 11: <使用されていない>.

ハードウェア パフォーマンス カウンターは、ハードウェア関連のアクティビティに関する統計情報を保存するための専用レジスタ セットです。低レベルのパフォーマンスまたは消費電力の分析を行うために使用されます。

R 1ビット
W 1ビット
BE 4ビット
MRRy 1ビット

メインメモリの制御信号：読み取り用R、書き込み用W、バイト選択用BE。
デフォルトでは、メモリは1クロックサイクルで読み書きします。メモリが数クロックサイクルに設定されている場合、メモリがR/W要求を終了するとMRdy信号が設定されます。

BE3=BW1、BE2=BW0、BE1=A1、BE0=A0

BEでは、ワード内の再読み取りするバイトと、それらをバス上のどこに配置するかを選択できます。

The central processing unit: Register File (RF).

注: このメカニズムは、テストのみを目的としています。マイクロコードの最終バージョンでこれらの信号を使用しないでください。.
RA、RB、RC、Cop の値は SelA (SelB、SelC、SelCop) と MR によって生成されるため、直接使用することは避けてください。 .

RA 5 bits

RB 5 bits

RC 5 bits

LC 1 bit

They allow you to select a register which value goes to A (RA) or B (RB), or to update the value of the selected record (RC). In the case of updating the register, in addition to using the RC signal, use the LC signal with a value of 1.

Figure 9 MUX A で利用可能なオプション


The central processing unit: Program Counter (PC)

M2 1 bit

They allow you to select which value can be used to update the program counter register (PC): adding four to the current PC content or the value that comes from the internal data bus.

Figure 10 Detail of the PC register management options

The central processing unit: Registers

C0, ..., C7 1 bit

It will store in the associated register, the value from the input of the register at the end of the clock cycle.

The central processing unit: Access to the data bus

Td 1 bit

Copy the value of the MBR register, after being transformed by the 'bytes selector' componente, into the data bus.

The central processing unit: Access to the address bus

Ta 1 bit

Copy the value from the MAR register into the address bus.

The central processing unit: Access to the internal data bus

T1, ..., T12 1 bit

Copy the value in the input of the associated tri-state element into the internal data bus, during the clock cycle.

The central processing unit: The ALU flags, plus I and U flags

NOTE: this mechanism is for testing purposes only. You should not use those signals in the final version of your microcode.
Please avoid to use TEST_C, ... directly because its value are generated by the ALU.

TEST_C, ..., TEST_U 1 bit

Let user to change on-the-fly the values of flags as they were generated by the hardware elements.
The current available flags are:

  • C: Carry
    • Add a b: (a >>> 31) && (b >>> 31)
  • V: oVerflow
    • Add a b: (a+b < 0) && (a >= 0) && (b >= 0) || (a+b >= 0) && (a < 0) && (b < 0)
  • N: Negative
    • Add a b: (a+b < 0)
  • Z: Zero
    • Add a b: (a+b == 0)

  • I: Interruption enabled/Interruption disabled
  • U: User level/Kernel level

ExCode 4 bits

Let microprogrammer to specify a inmediate value that should be copy into the internal data bus with signal T11.

The Input/Output Elements

IOR 1 bit
IOW 1 bit
IORdy 1 bit

Control signals for the I/O devices: IOR for read, IOW for write, and if the devices is asynchronous the IORdy signal is set when device ends the request.
